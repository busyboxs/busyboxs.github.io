argparse 库函数

发表于2022-09-29|更新于2022-09-29|cpp
|字数总计:1.1k|阅读时长:4分钟|阅读量:

argparse 是现代 C++ 程序参数解析库。该库是 header-only 的，使用该库需要使用 C++ 17。

argparse 支持位置参数，可选参数

位置参数语法支持：

  • _empty_: 参数命令为空
  • '-': 参数命令为 -
  • '-' decimal-literal: 参数命令为 - 加数字
  • !'-' anything: 不带前缀 - 的任何命令

ArgumentParse 类

constructor

构造函数构造一个参数解析器，解析器会根据传入的 add_args 参数设置对应默认参数。默认参数包括 help 和 version。对应的参数如下：

  • help: -h, --help
  • version: -v, --version

add_argument

添加参数。传入的是变长参数，是构造 Argument 类的第二个参数，即 std::array<std::string_view, N>&& a。

添加参数后，实际内部会构造一个 Argument 对象，然后保存到一个 map 中，key 是命令参数，value 是 Argument 对象。

add_parents

添加父级 ArgumentParser 对象的参数到该 ArgumentParser 对象中

add_description

添加参数描述

add_epilog

set_prefix_chars

设置可选参数的前缀

set_assign_chars

设置命令行中赋值字符，默认为 = 。

parse_args

解析参数

get

获取参数值，如果未设置则获取默认值，转换为对应的类型

present

获取参数值，如果未设置，则返回 std::optional 的空值 std::nullopt

is_used

对于用户提供了的可选参数时，返回 true，未提供则返回 false

is_subcommand_used

operator[]

获取指定命令行参数值

operator<<

输出流支持，用于打印

help

获取帮助信息，用于打印

usage

获取命令行使用说明，用于打印

print_help

add_subparser

添加子 ArgumentParser

Argument 类

Argument 类中的各个函数都是返回的 Argument&，所以 Argument 函数是支持链式调用的。

constructor

help

设置 --help 展示的帮助中的参数描述。

metavar

设置 metavar

default

设置参数的默认值

required

设置参数是否是必须的，如果不是必须的，则是可选的

implicit_value

设置参数的隐式值

action

设置参数执行的动作（函数）

append

设置参数是否是可重复的，支持多个值

scan

执行类型转换 action

nargs

通常情况下，一个命令行参数是与一个 action 进行关联的。nargs 可以将多个参数与一个 action 进行关联。

nargs 设置关联参数的个数范围或模式。模式支持 optional, any, at_least_one。各模式对应的范围如下：

  • optional: [0, 1]
  • any: [0, max]
  • at_least_one: [1, max]

remaining

consume

validate

get_inline_usage

get_arguments_length

operator<<

支持输出流，可用于打印
